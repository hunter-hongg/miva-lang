# 🎉 Miva v0.0.6 Released
>
> **Clairvoyant Falcon**

We are excited to announce the official release of **Miva Programming Language v0.0.6**!

This release focuses on **strengthening expressiveness, expanding standard capabilities, and laying the foundation for the toolchain**, moving Miva from "playable" toward "usable".

---

## 🔧 Core Updates

### 1. Control Flow Structures Are Here

Control flow logic is now available:

```miva
for i in (range(5, 10)) {
  printlns!("i is", i)
}
// Equivalent to
i := 5
while (i < 10) {
  printlns!("i is", i)
  i = i + 1 // Temporarily has a bug
}
// Equivalent to
i := 5
loop {
  printlns!("i is", i)
  i = i + 1 // Temporarily has a bug
  if (i >= 10) {
    // break // Will be supported in the future
  }
}
```

---

### 2. Module Syntax Update

In 0.0.6, the module definition syntax has been upgraded from `module "xxx"` to `module xxx`, making it easier for the checker to validate illegal formats.

```miva
// v0.0.5
module "foo.math"

// v0.0.6
module foo.math
```

This is a breaking change. Code will need simple rewriting.

---

### 3. Built-in Warnings

0.0.6 provides built-in deprecation warnings for some functions, such as `prints` (not a macro):

```miva
prints("Hello ", "World!") // Outputs a Warning, does not affect compilation
```

At the same time, 0.0.6 puts style checking into practice. When a module name is not in pure lowercase, or a variable/function name does not follow snake_case naming, the compiler will also output a Warning.

---

### 4. Better Error Management

Error management has been improved. All errors from a single file are now reported together:

```miva
// Example problematic code
// Missing module
main = () => {
  printx("Hello") // Function does not exist
  a = a + 1       // Variable does not exist
}
```

In 0.0.5, these errors would be reported one by one, hurting the debugging experience. In 0.0.6, all errors are displayed together, making it easier to fix them in one go.

---

### 5. Type Checking

0.0.6 promotes type checking to the Semantic stage:

```miva
module main
main = () => {
  a = 1 + true // 0.0.5 would silently succeed, 0.0.6 will report an error directly
}
```

There are still some known issues with type checking in this release, but 0.0.7 will bring better fixes and optimizations. Additionally, regardless of whether a function is marked `unsafe`, it must undergo type checking.

---

### 6. miver Released

Installing Miva is no longer a hassle:

```bash
bash -c "$(curl https://raw.githubusercontent.com/hunter-hongg/miver/script/install.sh)"
# Follow the wizard to install and activate
miver init   # Initialize Miver
miver list   # View available versions
miver switch v0.0.6   # Activate 0.0.6
```
